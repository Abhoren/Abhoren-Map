The Process

Inspiration

Much of the inspiration of this work comes from Loremap\'s Faerun map
A lot of ideas have been inspired from various works of fiction such as Lord of the Rings and Dungeons and Dragons.

What I did

In creating this site, I have utilized Leaflet to create markers that fully read json files to fill the sidebar that opens when clicked. I also created much of the content inside the markers, including the calendar, lore for several nations, and god pantheons.

What I learned

I especially learned from this project that I should always start earlier than I think I should. This project ended up being really fun, yet challenging. I definitely enriched my understanding of javascript and how to implement javascript libraries to make something really cool. I also learned that creating lore beforehand is much easier than writing while you are placing markers.

Struggles

I especially struggled on this project with filling a sidebar with information when the marker was clicked. After looking up tutorials that the TA sent me, I figured out on my own how to implement json files to fill the sidebars. This took me a period of a few days to work out, but once it started working, it was a relief. Another thing I struggled with on this project was creating lore on the spot for many locations, as we had not previously written about them. At the beginning we had only created lore for 5-10 markers, and I filled most of the others with lore. Finally I struggled with making the sidebars look better on mobile. For some reason, leaflet makes it very difficult to change the position of the sidebars in the CSS, and when you alter the width of the sidebars, much of the containing div of the sidebar disappears off the screen.

Above and Beyond

The original idea was to only have a few workable markers on the map. I believe the amount of markers and lore inside of each marker really shows how much work both me and Matthew have put into this. I learned how to read in jsons instead of just hard coding a few markers. We also created markers for all the other continents that display information that is helpful to the viewer about what they can one day expect in one of those areas. All in all, I believe we did our work well, and I look forward to five more continents.

The Process

Inspiration

In mid June of 2017 Conor, Nathan and myself started making a fantasy world based off of the Dungeons and Dragons campaign I was running. We have been writing lore and making maps for the world since then. I got the idea for this project one day when I was in the shower. I asked Conor if he wanted to make this project with me. The whole look and feel for the website was inspired by Loremap\'s Faerun map. That's where we found the leaflet.js library and the leaflet-sidebar.js plugin that was necessary to make this project work.

What I did

Most of my work was in making the tilesets for the pages. Once I got the tilesets working I was able to start adding cleaner functionality to the website. I added popup windows with brief summaries of information to each continent and major landmass on the main page and added the ability to move back and forth from the World Map to Odea. I also styled all the elements to be consistent. Finally I made the documentation page outlines and put them into two sidebars so the information was presented in a way that was similar to the rest of the website.

What I learned

I learned so much about how tilesets work and how the google maps api functions. Before this project I never thought how much work was put into just displaying a map let alone making it interactive. Before Conor and I dove into this project I knew next to nothing about javascript but now I have a much better understanding of how to implement scripts and write my own functions. Also libraries are now my best friend, without Leaflet.js this project would not have been possible.

Struggles

I struggled on many things for this project. The first massive roadblock for me was creating the tileset. When looking over Leaflet and Mapbox's(a leaflet plugin we didn't use) documentation all I could find was either premade tilesets from google maps or tilesets that had to me made with some odd vector filetypes. Luckily I found a tutorial that lead me to the photoshop-google-maps-tile-cutter and showed me how to load up a custom tileset in Leaflet. The second big thing I struggled with was the continent popup windows on the main page. Filling them with content wasn't too bad but when I tried to get the navigation button from the main page to Odea to work my progress ground to a halt. Popup windows take a string for content which can be edited with html tags, but because it's not actually an html file I wasn't able to get a reference to the button I created from outside of the popup window. After about 4 hours of trolling forums I learned I could just add the onclick property with a method call to the button tag and then my button worked perfectly. It is worth saying that I could have just styled a link tag to look like a button and have had no issues with it but I really wanted to get it to work with a button so I could learn more javascript.

Above and Beyond

I don't think there is too much visible above and beyond content on this website. A lot of what we learned and attempted and ultimately didn't use is what I would consider our above and beyond. I spent several hours getting a button to work where a link would have been fine just so I could get a better understanding of the Leaflet.js library. I spent many hours trolling through Leaflet.js and Leaf.js's css files just so I could learn how to style everything. And on top of all this we made the website expandable. So whenever we complete new lore and maps it is very easy to just add it in to the website. I did the best I could to make sure adding new content is as easy as possible.
